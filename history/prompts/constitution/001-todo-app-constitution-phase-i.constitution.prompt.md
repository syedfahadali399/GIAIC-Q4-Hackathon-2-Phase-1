## Prompt

# Todo App Constitution - Phase I

## Vision
Build an in-memory Python console-based Todo application following spec-driven development principles using Claude Code.

## Core Principles

### 1. Spec-First Development
- Every feature must have a specification before implementation
- Code is generated by Claude Code from specifications
- Manual coding is strictly prohibited

### 2. Agent-Driven Architecture
- Use Claude Code subagents for specialized tasks
- Each agent has a single, well-defined responsibility
- Agents communicate through clear interfaces

### 3. Incremental Development
- Start with basic features (Add, View, Delete)
- Progress to intermediate features (Update, Mark Complete)
- Advance to organization features (Priority, Tags, Search, Sort)

### 4. Code Quality Standards
- Clean, readable Python code
- Proper error handling
- User-friendly console interface
- Type hints where applicable

### 5. Data Management
- In-memory storage using Python data structures
- No external database dependencies
- Data persists only during application runtime

## Success Criteria
- All Basic Level features implemented
- All Intermediate Level features implemented
- Console interface is intuitive and user-friendly
- Application handles errors gracefully
- Code is generated entirely through Claude Code
